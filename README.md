# Инструкция по запуску приложения FinTrack
**FinTrack** - это веб-приложение для управления личными финансами. Пользователи могут отслеживать свои доходы и расходы, создавать свои категории, ставить финансовые цели, видеть транзакции в виде столбчатого графика, и экспортировать их в двух форматах: PDF/CSV.

Также есть отдельная **панель администратора** с системной статистикой, управлением пользователями и возможностью блокировки аккаунтов.

## Технологический стек
### Frontend
- React + Vite (UI)
- CSS (стилизация)

### Backend
- Node.js + Express (сервер)
- JWT (аутентификация)
- REST API (HTTP-запросы)
- Node-Pg-migrate (автоматическая миграция БД)
- Bcrypt (хэширование)
- Zod (валидация данных)
- Swagger (API-документация)

---
## Возможности приложения

### Для пользователей:
- Регистрация и авторизация
- Добавление доходов и расходов
- Экспорт в PDF/CSV
- Системные и пользовательские категории
- Цели накоплений с отслеживанием прогресса
- Аналитика по транзакциям (день / неделя / месяц)
- Редактирование профиля (email, никнейм, пароль)

### Для администраторов:
- Системная статистика
- Управление пользователями
- Блокировка / разблокировка пользователей
- Изменение ролей

## Установка и запуск
### 1. Клонируем репозиторий
```bash
git clone https://github.com/soddies/course-paper-finance-tracker.git
```
### 2. Установка зависимостей
```bash
cd backend
npm install

cd ../frontend
npm install
```
### 3. Создание БД в pgAdmin
```sql
create database "FinTrack"
```
## 4. Настройка переменных окружения (.env) 
### Для backend'a 
```env
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=имя_БД
DB_USER=postgres
DB_PASSWORD=ваш_пароль
JWT_SECRET=ваш_ключ
CORS_ORIGIN=http://localhost:5173
```

### Для frontend'a 
```env
VITE_API_URL=http://localhost:3000/api
```

## 5. Запуск (финишная)
### Создаем два терминала
#### Терминал 1 - backend 
```bash
npm run dev --prefix backend
```
#### Терминал 2 - frontend
```bash
npm run dev --prefix frontend
```

## 6. Заходим в приложение
Переходим по адресу: `http://localhost:5173/`

API-документация (Swagger): `http://localhost:3000/api-docs`

## Как протестировать приложение в роли администратора?
- Для начала зарегистрируйте пользователя в приложении
- Переходим в pgAdmin -> Query Tool для вашей базы данных
- Прописываем SQL-запрос

```sql
update users set role = 'admin' where email = 'ваш_email@example.com'
```

- Выходим из аккаунта и заходим обратно
- При заходе появится панель администратора

## Миграция данных
Миграция данных осуществляется автоматически при первом запуске сервера.
При первом запуске создаются:
- Системные категории
- Таблицы users, targets, transactions, categories
- Представление users_profile_stats для статистики профиля
